	Чтобы соединиться с сервером, приложению нужны две вещи:

Указать параметры подключения — так называемый DSN (сокращение от Data Source Name), 
	который может содержать в себе IP-адрес или Hostname сервера СУБД, 
		номер сетевого порта, название схемы (базы данных) и различные опции
Указать параметры авторизации — например, логин и пароль пользователя MySQL

	Структура DSN

Обычно DSN представляется строкой, которая содержит:
	Адрес сервера СУБД, например 172.16.0.7 или mysq-1.local
		Заметим, что IP-адрес 172.16.0.7 принадлежит диапазону приватных IP-адресов, 
			то есть не является допустимым адресом в сети Интернет
		Аналогично домен mysql-1.local находится в домене верхнего уровня local, 
			который является приватным; таким образом, домен mysql-1.local 
			невозможно зарегистрировать в сети Интернет
		Приватные адреса нужны, чтобы закрыть доступ к серверу СУБД из внешней сети 
			(из Интернета), настроив адресацию во внутренней сети дата-центра
	Номер сетевого порта или путь к сокету, например, :3306 для порта 3306
	Имя схемы (базы данных), например catboard для соединения с одноимённой базой данных
	Различные опции, специфичные для СУБД — например, параметры для установки определённой кодировки строк

	Пример DSN

Пример DSN: host=127.0.0.1;dbname=catboard
	здесь указано, что следует соединяться с сервером СУБД на машине с адресом 127.0.0.1 и затем открыть базу данных catboard
	порт не указан, значит, библиотека должна использовать порт по умолчанию, то есть 3306

	Вариации DSN

Многие библиотеки для работы с базами данных вносят в DSN две вариации:
	Имя протокола — указывается в начале строки DSN
		Например mysql: обычно означает «использовать MySQL protocol»
	Параметры авторизации — иногда их можно указывать прямо в строке DSN
		В этом случае строка DSN становится секретными данными
